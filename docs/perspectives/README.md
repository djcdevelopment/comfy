# Perspectives — the human lenses the problem was built from

This folder documents the **perspectives** Derek deliberately described the problem through, so that
(a) others can understand and copy the method, and (b) Derek can learn from his own building experience.

**Why perspectives, not data.** Derek withheld the data firehose (saves, sheets, 5 years of Discord)
and instead described the problem from every seat he'd sat in. *Data is infinite and directionless;
understanding is compressed and directional.* Building the lenses first is what made
[`../kernel.md`](../kernel.md) precise and the projections cheap. (See the kernel's method note.)

**How to read these.** Each file is one lens on the *same* kernel. Each documents: who the perspective
is, where it entered the conversation, what they see (fears · hopes · why they play · view of the
structure), what we learned through them, how the understanding evolved, and what it changed in the
kernel/build. If two files seem to describe the same disease from different angles — that's the point.
The kernel is the thing they all resolve to.
