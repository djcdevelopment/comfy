# Community systems framework

This small directory states the rules underneath the repo's community-facing work.

- Read [`AGENTS.md`](AGENTS.md) for the concrete capture → structure → validate → render → version
  workflow and its safety boundaries.
- Read [`PHILOSOPHY.md`](PHILOSOPHY.md) for why tools here must be open, craftable, repairable, and
  owned by the people using them.
- Continue to [`../recipes/README.md`](../recipes/README.md) for runnable examples.

The central constraint is simple: standardize plumbing, never a community's content. Preserve raw
sources, flag uncertainty, render back into formats people already use, and leave enough homework for
the next person to repair the process without its author.
